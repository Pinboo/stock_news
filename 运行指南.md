# 🚀 运行指南

## 前置要求

### 1. 安装 Python

你需要先安装 Python 3.8 或更高版本。

#### Windows 系统安装 Python：

**方法一：从官网下载（推荐）**
1. 访问：https://www.python.org/downloads/
2. 下载最新的 Python 3.x 版本（如 Python 3.11）
3. 运行安装程序
4. ⚠️ 重要：勾选 "Add Python to PATH"
5. 点击 "Install Now"

**方法二：从 Microsoft Store 安装**
1. 打开 Microsoft Store
2. 搜索 "Python 3.11"
3. 点击安装

**验证安装：**
打开命令行（PowerShell 或 CMD），输入：
```bash
python --version
```
应该显示类似：`Python 3.11.x`

---

## 运行步骤

### 第一步：安装依赖

在项目目录下打开命令行，运行：

```bash
pip install -r requirements.txt
```

这会安装所有需要的 Python 库：
- akshare（股票数据）
- requests（网络请求）
- schedule（定时任务）
- python-dotenv（环境变量）
- openai（AI 接口）
- pandas（数据处理）

安装时间约 1-3 分钟，请耐心等待。

---

### 第二步：测试钉钉机器人

确保你已经在 `.env` 文件中配置好钉钉信息，然后运行：

```bash
python test_dingtalk.py
```

**预期结果：**
- 控制台显示 "✅ 钉钉消息发送成功"
- 钉钉群里收到测试消息

**如果失败：**
- 检查 `.env` 文件中的 `DINGTALK_WEBHOOK` 是否正确
- 检查机器人安全设置（关键词要包含"A股"）
- 查看 `DINGTALK_SETUP.md` 排查问题

---

### 第三步：测试完整功能

运行完整的分析流程（立即执行一次）：

```bash
python test_run.py
```

**这个脚本会：**
1. 📰 抓取最新财经新闻
2. 📊 获取大盘实时行情
3. 📈 获取你关注的个股数据
4. 🤖 AI 分析大盘和个股
5. 📤 推送到钉钉群

**预期结果：**
- 控制台显示各步骤的执行情况
- 钉钉群收到完整的分析报告

**如果失败：**
- 检查 AI 模型配置（API Key、Base URL、Model Name）
- 检查网络连接
- 查看控制台的错误信息

---

## 常见问题

### Q1: pip 命令不存在
**错误信息**：`pip: 无法将"pip"项识别为 cmdlet`

**解决方案：**
1. 确认 Python 已安装：`python --version`
2. 如果 Python 已安装但 pip 不可用，尝试：
   ```bash
   python -m pip install -r requirements.txt
   ```
3. 如果还不行，重新安装 Python 并勾选 "Add Python to PATH"

### Q2: 安装依赖失败
**错误信息**：`ERROR: Could not find a version...`

**解决方案：**
1. 升级 pip：
   ```bash
   python -m pip install --upgrade pip
   ```
2. 重新安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

### Q3: 获取股票数据失败
**错误信息**：`获取股票数据失败`

**可能原因：**
- 网络问题
- akshare 库版本问题
- 股票代码错误

**解决方案：**
1. 检查网络连接
2. 更新 akshare：
   ```bash
   pip install --upgrade akshare
   ```
3. 检查 `.env` 中的股票代码是否正确

### Q4: AI 分析失败
**错误信息**：`AI分析失败`

**可能原因：**
- API Key 错误
- 账户余额不足
- 网络无法访问 API

**解决方案：**
1. 检查 `.env` 中的 API 配置
2. 确认账户有余额
3. 测试网络连接：
   ```bash
   ping api.openai.com
   ```
   或
   ```bash
   ping dashscope.aliyuncs.com
   ```

### Q5: 钉钉消息发送失败
**错误信息**：`钉钉消息发送失败: errcode: 310000`

**解决方案：**
- 检查机器人安全设置
- 确保消息包含关键词（系统已自动包含"A股"）
- 查看 `DINGTALK_SETUP.md` 详细排查

## 进阶使用

### 修改推送时间
编辑 `.env` 文件：
```env
# 每天推送3次：开盘前、午盘、收盘后
PUSH_TIME=09:00,12:00,15:30
```

### 修改关注股票
编辑 `.env` 文件：
```env
# 添加更多股票
STOCK_POOL=000001,600519,000858,601318,600036,000333
```

### 后台运行（Linux/Mac）
```bash
# 使用 nohup 后台运行
nohup python main.py > output.log 2>&1 &

# 查看日志
tail -f output.log

# 停止程序
ps aux | grep main.py
kill <进程ID>
```

### Windows 后台运行
1. 创建批处理文件 `start.bat`：
```batch
@echo off
python main.py
pause
```

2. 双击运行，或使用任务计划程序开机自启

## 性能优化

### 减少 API 调用
如果 AI 分析费用较高，可以：
1. 只在收盘后分析（修改 PUSH_TIME）
2. 减少关注的股票数量
3. 使用更便宜的模型（如 hunyuan-lite、deepseek-chat）

### 提高稳定性
1. 使用国内 AI 模型（通义千问、混元）
2. 配置多个推送时间作为备份
3. 定期检查日志
