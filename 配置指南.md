# 📋 配置指南

## 重要说明

- ✅ `.env` - 这是你要填写真实配置的文件（已创建）
- ℹ️ `.env.example` - 这只是配置模板，不要在这里填写

`.env` 文件已经被 `.gitignore` 忽略，不会被提交到代码仓库，保护你的密钥安全。

---

## 配置步骤

### 第一步：选择钉钉接入方式（二选一）

#### 方式1：自定义机器人 ⭐ 推荐新手

1. 打开钉钉，进入你想接收消息的群聊
2. 点击右上角 `···` → `智能群助手` → `添加机器人`
3. 选择 `自定义` 机器人
4. 填写机器人名称：`A股智能分析助手`
5. 安全设置选择 `自定义关键词`，填写：`A股`
6. 复制生成的 Webhook 地址
7. 打开 `.env` 文件，填写：

```env
DINGTALK_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=你复制的token
```

#### 方式2：企业内部应用（需要管理员权限）

1. 访问 https://open-dev.dingtalk.com/
2. 创建企业内部应用
3. 获取 AppKey、AppSecret、AgentId
4. 打开 `.env` 文件，填写：

```env
DINGTALK_APPKEY=你的AppKey
DINGTALK_APPSECRET=你的AppSecret
DINGTALK_AGENT_ID=你的AgentId
```

详细步骤查看：`DINGTALK_SETUP.md`

---

### 第二步：配置AI模型

#### 选项1：使用 OpenAI（需要国际网络）

```env
OPENAI_API_KEY=sk-你的OpenAI密钥
OPENAI_BASE_URL=https://api.openai.com/v1
MODEL_NAME=gpt-4
```

#### 选项2：使用通义千问 ⭐ 推荐国内用户

1. 访问 https://dashscope.console.aliyun.com/
2. 开通服务并创建 API Key
3. 填写配置：

```env
OPENAI_API_KEY=sk-你的通义千问密钥
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MODEL_NAME=qwen-max
```

#### 选项3：使用腾讯混元 ⭐ 推荐腾讯云用户

1. 访问 https://console.cloud.tencent.com/hunyuan
2. 开通服务并获取 SecretId 和 SecretKey
3. 填写配置：

```env
OPENAI_API_KEY=你的SecretId:你的SecretKey
OPENAI_BASE_URL=https://api.hunyuan.cloud.tencent.com/v1
MODEL_NAME=hunyuan-lite
```

注意：混元的 API Key 格式为 `SecretId:SecretKey`（用冒号连接）

#### 选项4：使用 DeepSeek（最便宜）

1. 访问 https://platform.deepseek.com/
2. 注册并获取 API Key
3. 填写配置：

```env
OPENAI_API_KEY=sk-你的DeepSeek密钥
OPENAI_BASE_URL=https://api.deepseek.com/v1
MODEL_NAME=deepseek-chat
```

更多模型配置查看：`MODEL_CONFIG.md`

### 第三步：配置股票池（可选）

在 `.env` 文件中修改 `STOCK_POOL`：

```env
# 示例：关注4只股票
STOCK_POOL=000001,600519,000858,601318
```

常见股票代码：
- 000001 - 平安银行
- 600519 - 贵州茅台
- 000858 - 五粮液
- 601318 - 中国平安
- 600036 - 招商银行
- 000333 - 美的集团

---

### 第四步：配置推送时间（可选）

在 `.env` 文件中修改 `PUSH_TIME`：

```env
# 示例：开盘后和收盘后各推送一次
PUSH_TIME=09:00,15:30
```

## 常见问题

### Q1: 找不到 .env 文件？
A: 在项目根目录下，`.env` 文件已经创建好了。如果看不到，可能是被隐藏了（文件名以 `.` 开头）。

### Q2: 配置后不生效？
A: 确保：
1. 配置填写在 `.env` 文件（不是 `.env.example`）
2. 配置项后面的 `=` 号后直接跟值，不要有空格
3. 不要用引号包裹值

### Q3: 钉钉消息发送失败？
A: 检查：
1. Webhook 地址是否完整复制
2. 机器人安全设置（关键词要包含"A股"）
3. 消息内容是否包含设置的关键词

### Q4: AI分析失败？
A: 检查：
1. API Key 是否正确
2. Base URL 是否正确
3. 账户余额是否充足
4. 网络是否能访问API地址
