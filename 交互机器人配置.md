# 交互机器人配置指南

## 为什么需要特殊配置？

钉钉机器人有两种工作模式：

### 1. 主动推送模式 ✅（当前已实现）
- 机器人主动发送消息到群
- 不需要回调地址
- 不需要云服务器
- 适合：定时推送、通知

### 2. 交互问答模式 🔄（需要额外配置）
- 用户在群里@机器人提问
- 机器人接收消息并回复
- 需要钉钉能把消息发给你的程序

---

## 实现交互问答的三种方案

### 方案一：命令行测试模式 ⭐ 最简单

直接在命令行测试机器人的问答功能，不需要任何配置。

**使用方法：**
```bash
python interactive_bot.py
```

**效果：**
```
👤 你: 000001
🤖 机器人:
📈 平安银行(000001)
💰 当前价格: 12.50元
📈 涨跌幅: +2.5%
...
```

**优点：**
- 无需配置
- 立即可用
- 适合测试功能

**缺点：**
- 不能在钉钉群里用
- 只能在电脑上测试

---

### 方案二：群机器人 Outgoing 模式 ⭐⭐ 推荐

使用钉钉群机器人的 Outgoing 功能，需要一个公网地址。

**配置步骤：**

1. **创建 Outgoing 机器人**
   - 钉钉群 → 智能群助手 → 添加机器人 → 自定义
   - 选择「Outgoing」类型
   - 填写 POST 地址：`http://你的公网IP:8080/webhook`
   - 设置 Token（用于验证）

2. **启动服务器**
   ```bash
   python bot_server.py
   ```

3. **使用内网穿透（如果没有公网IP）**
   
   使用 ngrok 或 花生壳：
   ```bash
   # 使用 ngrok
   ngrok http 8080
   
   # 会得到一个公网地址，如：
   # https://abc123.ngrok.io
   ```
   
   然后在钉钉机器人配置中填写：
   ```
   https://abc123.ngrok.io/webhook
   ```

**优点：**
- 可以在钉钉群里用
- 支持多人同时使用
- 功能完整

**缺点：**
- 需要公网地址或内网穿透
- 需要保持服务器运行

---

### 方案三：企业应用 Stream 模式 ⭐⭐⭐ 最强大

使用钉钉企业应用的 Stream 模式，不需要公网地址。

**配置步骤：**

1. **开通 Stream 模式**
   - 进入钉钉开发者后台
   - 应用开发 → 你的应用 → 开发配置
   - 开通「Stream 模式」

2. **安装 SDK**
   ```bash
   pip install dingtalk-stream
   ```

3. **修改代码使用 Stream SDK**
   ```python
   from dingtalk_stream import AckMessage
   import dingtalk_stream
   
   # 详细代码见官方文档
   ```

**优点：**
- 不需要公网地址
- 不需要内网穿透
- 官方推荐方案
- 稳定可靠

**缺点：**
- 配置较复杂
- 需要企业管理员权限

**官方文档：**
https://open.dingtalk.com/document/orgapp/stream-mode-overview

## 常见问题

### Q1: 为什么不能直接在钉钉群里用？
A: 钉钉需要知道把消息发到哪里，有三种方式：
- Outgoing：需要你提供一个网址
- Stream：使用钉钉的长连接
- 命令行：只能在电脑上测试

### Q2: 我没有云服务器怎么办？
A: 使用内网穿透工具：
- ngrok（国外）
- 花生壳（国内）
- frp（开源）

### Q3: 内网穿透安全吗？
A: 
- 使用 HTTPS
- 配置 Token 验证
- 不要暴露敏感信息

### Q4: Stream 模式怎么配置？
A: 参考钉钉官方文档：
https://open.dingtalk.com/document/orgapp/stream-mode-overview
